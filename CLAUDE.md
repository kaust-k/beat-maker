# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Project Is

A camera-based hardware beat sequencer. Colored tiles placed on a physical grid are detected via a USB camera using computer vision, and the positions trigger drum sounds in real time. The grid acts as an 8-beat × 6-track step sequencer.

## Running the App

```bash
pip install -r requirements.txt
python -m src.main
```

Must be run from the project root — it reads `config.yaml` and `calibration.json` from the current working directory.

There is no build step, test suite, or linter configured.

## Keyboard Controls

| Key | Action |
|-----|--------|
| `c` | Start calibration (then click 4 grid corners in order: TL, TR, BR, BL) |
| `b` | Capture background lighting baseline (run with no tiles present) |
| `space` | Pause / play |
| `+` / `-` | Tempo ±5 BPM |
| `r` | Reset beat position to column 0 |
| `q` | Quit |

## Architecture

```
main.py
  ├── CameraThread        reads frames from USB camera (daemon thread)
  ├── GridMapper          perspective transform + calibration I/O
  ├── PresenceTileDetector + SelectorDetector   HSV saturation tile detection
  ├── GridState           debounced 2D tile presence matrix (thread-safe)
  ├── BeatSequencer       fires columns at BPM intervals (daemon thread)
  ├── ProfileManager      queues/applies profile switches at cycle boundaries
  └── AudioPlayer         preloads all sounds at startup; one channel per track
```

**Detection pipeline:** `CameraThread` → `GridMapper.warp()` → `PresenceTileDetector` → `GridState.update()` → `BeatSequencer` reads state → `AudioPlayer.play(row)`

**Vision approach:** HSV saturation delta thresholding (color-agnostic). Per-cell baseline S-channel (captured via `b` key) is subtracted to adapt to ambient lighting. Temporal hysteresis (`min_present_frames` / `min_absent_frames`) debounces detections.

**Threading model:** Camera and sequencer run as daemon threads. Main thread owns the OpenCV window and keyboard/mouse input. `GridState` uses a lock. Profile and tempo changes use GIL-safe int assignment.

**Profile switching:** `ProfileManager.request_switch()` queues a change; it's applied by `on_cycle_end()` callback (when beat wraps to column 0) to avoid mid-beat artifacts. All profiles are preloaded at startup.

**Audio:** `miniaudio` decodes files to int16 stereo 44.1 kHz; pygame mixer plays them. Falls back to numpy synthesis for kick, snare, hihat, clap, tom, cymbal if file is missing. RMS-normalized to −18 dBFS at load time.

## Key Configuration

`config.yaml` controls:
- Grid dimensions (default 8 beats × 6 tracks)
- Tempo (default 180 BPM, range 20–300)
- Camera ID and optional rotation (0/90/180/270°)
- Detection thresholds: `saturation_delta` (40), tile coverage hysteresis (15% present / 5% absent), debounce frame counts
- 5 sound profiles ("Kit A"–"Kit E"), each with 6 tracks specifying audio file paths under `assets/sounds/` and per-track volume

`calibration.json` is auto-generated by the app — stores 4 corner pixel coords, the 3×3 perspective warp matrix, and a per-cell baseline S-channel array. Do not hand-edit.

`config.local.yaml` (gitignored) can override any values from `config.yaml` — only the keys you specify are merged on top; everything else falls through to the defaults. Useful for per-machine settings like `camera_id` or `camera_rotate`.

```yaml
# example config.local.yaml
camera_id: 0
tempo_bpm: 120.0
```
